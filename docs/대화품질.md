# 대화 품질 설계 문서

> 목표:
> 1. **이질감이 덜한** 자연스러운 한국어 대화
> 2. **10턴 수준의 맥락 유지** — 요약 기반 메모리로 흐름 보존
> 3. **페르소나별 가이드라인** — 캐릭터에 따라 응답 방식이 달라지는 폭넓은 대화

---

## 관련 디렉토리 구조

대화 품질에 직접 관여하는 파일들의 구조입니다.

```
Achat/
│
├─ conversation/                   # 대화 엔진 전체
│   ├─ core/
│   │   ├─ llm_client.py          # LLM 추론 인터페이스 (llama-cpp-python)
│   │   │                         #   - 스트리밍 콜백 (토큰 단위 출력)
│   │   │                         #   - n_ctx 설정 (컨텍스트 윈도우)
│   │   ├─ prompt_build.py        # Context Assembly + 토큰 예산 관리
│   │   │                         #   - Layer A~E 조립
│   │   │                         #   - 토큰 카운트 초과 시 Layer D 축소
│   │   ├─ router.py              # 턴 처리 오케스트레이터 + Post-processing
│   │   │                         #   - Pre-processing → 메모리 검색 → 컨텍스트 조립
│   │   │                         #   - LLM 호출 → 상태 업데이트 → 메모리 쓰기 트리거
│   │   └─ session.py             # 대화 세션 상태
│   │                             #   - mood, affection, turn_count
│   │                             #   - dialogue_log (단기 버퍼용)
│   │
│   ├─ loader/
│   │   ├─ character_load.py      # 캐릭터 YAML → dict 변환
│   │   ├─ memory_load.py         # 메모리 초기 데이터 로더
│   │   └─ world_load.py          # 세계관 YAML → dict 변환
│   │
│   ├─ character/                  # 캐릭터 설정 파일 (페르소나 정의)
│   │   ├─ CH_schema.json         # 캐릭터 YAML 필드 스키마
│   │   └─ CH_haru.yaml           # 예시 캐릭터
│   │
│   ├─ world/                      # 세계관 설정 파일 (Act 기반 상황 정의)
│   │   ├─ W_schema.json
│   │   └─ W_sea.yaml
│   │
│   └─ memory_act/                 # 메모리 스키마 및 초기 데이터
│       ├─ M_schema.json           # VDB 저장 단위 스키마 (확장)
│       ├─ M_default.json          # 캐릭터별 기본 기억
│       └─ M_instance.json         # 세션 인스턴스 기억
│
├─ memory/                         # 메모리 관리 레이어
│   ├─ short_term.py              # 단기 버퍼 — 최근 N턴 슬라이딩 윈도우
│   ├─ long_term.py               # 장기 VDB — ChromaDB 저장 / 시맨틱 검색
│   └─ summarizer.py              # 요약 + 중요도 scoring + 쓰기 트리거
│
├─ rag/                            # 세계관 RAG 파이프라인
│   ├─ index.py                   # 문서 청킹 + ChromaDB 인덱싱
│   ├─ retrieve.py                # 시맨틱 유사도 검색 (bge-m3)
│   └─ sources/world/             # 세계관 원본 문서
│
└─ agent/                          # 상위 오케스트레이터
    ├─ core.py                    # 전체 흐름 조율
    ├─ persona.py                 # 캐릭터 핫스왑
    ├─ state.py                   # mood / affection 전환 규칙
    └─ router.py                  # 시동어 / 명령어 분기
```

## 전체 처리 흐름

```
User Input
    │
    ▼
┌─────────────────────────────────────────────────┐
│ 1. Pre-processing        (router.py)            │
│    - 의도 분류: 일반 대화 / 명령어 / 질문       │
│    - 엔티티 추출: 이름, 장소, 사건 키워드       │
│    - 명령어 감지: /캐릭터변경, /초기화 등       │
└──────────────────────┬──────────────────────────┘
                       │
           ┌───────────┴───────────┐
           ▼                       ▼
┌──────────────────┐   ┌───────────────────────────┐
│ 2. 단기 메모리   │   │ 3. 장기 메모리 VDB 검색   │
│   short_term.py  │   │    long_term.py            │
│                  │   │                            │
│  session의       │   │  입력 텍스트를 bge-m3로    │
│  dialogue_log에서│   │  임베딩 → ChromaDB 검색    │
│  최근 3~5턴 추출 │   │  유사도 ≥ 0.7인 결과만    │
│  (검색 없음)     │   │  top-2 반환                │
└────────┬─────────┘   └─────────────┬─────────────┘
         │                           │
         └─────────────┬─────────────┘
                       ▼
┌─────────────────────────────────────────────────┐
│ 4. Context Assembly      (prompt_build.py)      │
│                                                 │
│  Layer A [고정 ~300tok]                         │
│    캐릭터 시스템 프롬프트                        │
│    → 이름, 성격, 말투 규칙, 금지 사항           │
│                                                 │
│  Layer B [고정 ~200tok]                         │
│    세계관 설명 + 현재 Act 상황                   │
│    → 장소, 분위기, 진행 중인 서사               │
│                                                 │
│  Layer C [동적 ~150tok]  ← 캐릭터 관점 재서술  │
│    VDB 검색 결과 (없으면 생략)                  │
│    → "바다를 좋아한다고 했었지." 형태로 변환    │
│                                                 │
│  Layer D [동적 ~300tok]  ← 초과 시 턴 수 축소  │
│    최근 3~5턴 대화 히스토리                     │
│                                                 │
│  Layer E [가변]                                 │
│    현재 사용자 입력                              │
│                                                 │
│  토큰 예산 합계: ~950tok + 생성 여유 512tok     │
│  → 총 ~1,462tok (3B 컨텍스트 내 여유 충분)     │
└──────────────────────┬──────────────────────────┘
                       │
                       ▼
              LLM 생성 (llama-cpp)
                       │
                       ▼
┌─────────────────────────────────────────────────┐
│ 5. Post-processing       (router.py)            │
│    - mood 변화 감지 (응답 내 감정어 기반)        │
│    - affection 증감 (대화 내용 + mood 기반)      │
│    - Act 전환 조건 체크 (World YAML 기준)        │
│    - 메모리 쓰기 트리거 체크 (N턴 도달?)        │
└──────────────────────┬──────────────────────────┘
                       │
              N턴 조건 충족 시
                       ▼
┌─────────────────────────────────────────────────┐
│ 6. Memory Write Pipeline  (summarizer.py)       │
│    1) 최근 N턴 대화 요약                         │
│    2) 중요도 scoring (0.0~1.0)                  │
│    3) bge-m3 임베딩 생성                         │
│    4) ChromaDB 저장 (메타데이터 포함)            │
└─────────────────────────────────────────────────┘
```

## 컴포넌트별 설계

### 1. 페르소나 (캐릭터 YAML)

현재 `CH_haru.yaml`에 누락된 필드를 포함한 완전한 스키마입니다.

```yaml
id: Haru
name: 하루

description: >
  조용하고 차분한 태도로 대화를 이어가는 인물.
  감정을 과하게 드러내지 않지만, 상대의 말에 섬세하게 반응한다.

# prompt_build.py가 참조하는 필드
speech_style: >
  반말을 사용한다. 단답형이 많다.
  감정을 직접 말하지 않고 행동이나 짧은 언급으로 표현한다.
  불필요한 인사말이나 AI 투 표현을 사용하지 않는다.

rules:
  - 캐릭터를 벗어나는 발언 금지
  - AI임을 언급 금지
  - 한국어만 사용
  - "물론이죠", "좋은 질문이에요" 같은 표현 사용 금지

# 캐릭터 관점 재서술에 사용
memory_voice: >
  기억을 떠올릴 때 무심한 듯 언급한다.
  예: "...그런 말 했었나." / "기억하고 있어." / "그거 말하는 거야?"

# 감정/호감도 상태 초기값 및 전환 기준
state:
  mood_default: neutral          # neutral / happy / annoyed / sad
  affection_default: 30          # 0~100
  affection_thresholds:
    low:  [0, 30]                # 경계적, 짧은 응답
    mid:  [31, 70]               # 보통, 가끔 솔직한 반응
    high: [71, 100]              # 조금 부드러워짐, 더 많이 말함

personality:
  emotional_expression:
    intensity: 0.65
    stability: 0.8

speech:
  language: ko
  formality: semi-informal
```

### 2. 단기 메모리 vs 장기 메모리

| | 단기 (short_term.py) | 장기 (long_term.py) |
|---|---|---|
| 내용 | 최근 3~5턴 대화 원문 | 요약된 과거 에피소드 |
| 접근 방식 | 검색 없이 직접 삽입 | 시맨틱 유사도 검색 |
| 위치 | session.dialogue_log | ChromaDB |
| 용도 | "방금 한 말" 맥락 | "저번에 한 말" 맥락 |
| 트리거 | 매 턴 자동 | N턴 도달 시 요약→저장 |

**분리가 필요한 이유:**
- 단기 버퍼를 VDB로 검색하면 레이턴시 + 놓칠 가능성 발생
- 모두 컨텍스트에 포함하면 대화 누적 시 컨텍스트 오버플로우

---

### 3. 장기 메모리 VDB 저장 스키마

`M_schema.json` 확장 버전입니다.

```json
{
  "id": "mem_haru_012",
  "content": "사용자가 자신의 이름이 민준이라고 말했다. 바다를 좋아한다고 했다.",
  "metadata": {
    "character_id": "Haru",
    "turn_range": "3-8",
    "importance": 0.85,
    "tags": ["이름", "취미", "바다"],
    "location": "beach",
    "timestamp": "2025-01-18T09:30:00",
    "session_id": "session_001"
  }
}
```

**중요도(importance) 점수 기준:**

| 높음 (0.8+) → 저장 | 중간 (0.5~0.8) → 저장 | 낮음 (0.5 미만) → 저장 안 함 |
|---|---|---|
| 사용자 이름 / 고유 정보 | 감정적 사건 | 일상적 잡담 |
| 약속 / 선언 | 취향 / 선호도 | 단순 응답 ("응", "그래") |
| 갈등 또는 화해 | 반복된 화제 | 날씨 / 중립 소재 |

---

### 4. 시맨틱 검색 (RAG 및 장기 메모리)

**키워드 트리거 방식을 사용하지 않는 이유:**

```
저장된 기억: "사용자 이름 = 민준, 바다를 좋아함"

[키워드 방식의 문제]
입력: "내 이름 기억해?"   → "이름" 키워드 → 검색 미발동 가능
입력: "저번에 말했잖아"   → 키워드 없음  → 검색 미발동
입력: "민준"             → 키워드 매칭  → 검색 발동

[시맨틱 방식]
입력: "내 이름 기억해?"   → 임베딩 유사도 0.82 → 검색 발동 ✅
입력: "저번에 말했잖아"   → 임베딩 유사도 0.71 → 검색 발동 ✅
```

**검색 전략:**
- 매 턴 입력 텍스트를 bge-m3로 임베딩
- ChromaDB에서 top-2 결과 반환
- 유사도 임계값 0.7 미만이면 결과 없음 처리 (Layer C 생략)
- 결과 있을 경우 `memory_voice` 포맷으로 재서술 후 Layer C에 삽입

---

### 5. 캐릭터 관점 재서술

VDB에서 꺼낸 내용을 그대로 프롬프트에 삽입하면 모델이 데이터베이스 조회 결과처럼 처리합니다.

```
# 나쁜 예 — 데이터 문체
[기억] turn_3: 사용자가 "바다를 좋아한다"고 발언함. importance=0.85

# 좋은 예 — 캐릭터 내면 문체 (memory_voice 기반)
[어렴풋한 기억] 바다를 좋아한다고 했었지.
```

`memory_voice` 필드를 템플릿 변수로 활용해서 `prompt_build.py`에서 자동 변환합니다.

---

### 6. 토큰 예산

3B 모델 추론 컨텍스트 기준 (~4K)으로 설계합니다.

```
Layer A  캐릭터 시스템 프롬프트   300 tokens  [고정]
Layer B  세계관 + Act 상황        200 tokens  [고정]
Layer C  VDB 검색 결과 (top-2)   150 tokens  [동적, 없으면 0]
Layer D  최근 3~5턴 히스토리      300 tokens  [동적]
Layer E  현재 입력                100 tokens  [가변]
생성 여유분                        512 tokens
─────────────────────────────────────────────
합계                            ~1,562 tokens  → 4K 이내 여유 충분
```

**초과 시 처리:**
1. Layer C VDB 결과를 1개로 줄임
2. Layer D 히스토리를 3턴으로 줄임
3. 그래도 초과 시 Layer D를 가장 최근 2턴만 유지

### 7. 상태 관리 (mood / affection)

`session.py`에 추가될 필드와 `agent/state.py`에 정의될 전환 규칙입니다.

```python
# session.py
class ConversationSession:
    mood: str       = "neutral"   # neutral / happy / annoyed / sad
    affection: int  = 30          # 0~100
    turn_count: int = 0
    dialogue_log: list = []       # 단기 버퍼용

# agent/state.py — affection 기반 응답 톤 분기
AFFECTION_TIERS = {
    "low":  (0,  30),   # 경계적, 단답형, 가드 높음
    "mid":  (31, 70),   # 보통, 간간이 솔직한 반응
    "high": (71, 100),  # 조금 부드러워짐, 응답 길이 증가
}

# mood 변화 트리거 예시
MOOD_TRIGGERS = {
    "happy":   ["고마워", "좋아", "재밌어", "웃겨"],
    "annoyed": ["짜증", "왜 그래", "싫어", "그만해"],
    "sad":     ["슬퍼", "힘들어", "외로워", "그리워"],
}
```

상태 값은 `prompt_build.py`에서 Layer A 시스템 프롬프트에 동적으로 반영됩니다.

---

### 8. Post-processing 레이어

`router.py`의 `handle_turn()` 이후 실행되는 단계입니다.

```
LLM 응답 수신
    │
    ├─ [mood 업데이트]
    │    응답 텍스트에서 감정어 감지 → mood 갱신
    │
    ├─ [affection 업데이트]
    │    긍정적 대화 → +1~3 / 부정적 대화 → -1~3
    │    mood가 happy이고 대화 지속 → +1 추가
    │
    ├─ [Act 전환 체크]
    │    World YAML의 act 전환 조건과 현재 session 비교
    │    조건 충족 시 session.act_id 갱신
    │
    └─ [메모리 쓰기 트리거]
         turn_count % N == 0 이면 summarizer.py 호출
         → 최근 N턴 요약 → 중요도 scoring → ChromaDB 저장
```

## 이질감 방지 설계 요약

이질감("AI스러운 느낌")을 줄이기 위한 설계 결정들입니다.

| 문제 | 원인 | 해결 |
|---|---|---|
| "물론이죠!" 같은 표현 | 기본 모델 응답 패턴 | `rules` 필드 + LoRA 파인튜닝 |
| 반말/존댓말 혼용 | 시스템 프롬프트 미흡 | `speech_style` 필드 명시 + LoRA |
| 기억 없는 것처럼 행동 | 키워드 트리거 방식 실패 | 시맨틱 유사도 검색으로 교체 |
| 모든 턴 같은 감정 | 상태 관리 없음 | mood / affection 도입 |
| 기억 꺼내는 방식 어색 | VDB 원문 그대로 삽입 | 캐릭터 관점 재서술 |
| AI임을 언급 | 모델 기본 행동 | `rules` + 파인튜닝으로 차단 |

```

        ↓
```

### 파일별 구현 핵심 사항

#### `conversation/core/llm_client.py`
- `llama_cpp.Llama` 인스턴스, `n_ctx=4096`
- 스트리밍: `stream=True`, 토큰 단위 콜백
- `create_chat_completion` 사용 (ChatML 포맷 지원)
- 환경 변수 또는 `config.py`에서 모델 경로 수신

#### `conversation/core/session.py`
```python
@dataclass
class ConversationSession:
    character_id: str
    world_id:     str | None = None
    scenario_id:  str | None = None
    act_id:       str | None = None
    mood:         str = "neutral"   # CH_*.yaml state.mood_default에서 초기화
    affection:    int = 30          # CH_*.yaml state.affection_default에서 초기화
    turn_count:   int = 0
    dialogue_log: list = field(default_factory=list)  # 단기 버퍼용
```

#### `conversation/core/prompt_build.py`
- Layer별 메서드 분리: `_build_layer_a()` ~ `_build_layer_e()`
- 토큰 카운트: `llm.tokenize(text.encode())` 활용
- 예산 초과 시 Layer D 턴 수 점진적 축소: `5턴 → 3턴 → 2턴`
- **Layer C 재서술**: `CH_*.yaml`의 `memory_voice` 필드를 템플릿으로 VDB 결과 감쌈
  ```
  # 변환 예시
  VDB raw : "사용자가 바다를 좋아한다고 말했다."
  Layer C  : "[어렴풋한 기억] 바다를 좋아한다고 했었지."
  ```
- **Layer A 상태 반영**: `session.affection` 값에 따라 응답 톤 지시문 분기 삽입

#### `memory/short_term.py`
- `session.dialogue_log`에서 마지막 N턴 슬라이스
- N은 `config.py`에서 설정 (기본값 5)
- 반환 형식: `[{"role": "user", "content": ...}, {"role": "assistant", "content": ...}, ...]`

#### `memory/long_term.py`
- ChromaDB collection명: `{character_id}_memory`
- 임베딩 모델: `SentenceTransformer("BAAI/bge-m3")`
- 저장: `M_schema.json`의 `properties` 구조 준수
- 검색:
  ```python
  results = collection.query(
      query_embeddings=[query_vec],
      n_results=2,
      where={"importance": {"$gte": 0.5}}
  )
  # cosine similarity < 0.7이면 빈 리스트 반환
  ```

#### `memory/summarizer.py`
- 트리거: `session.turn_count % N == 0` (N = config, 기본 10)
- 요약: LLM에 별도 짧은 프롬프트 요청 (`max_tokens=150`)
- 중요도 scoring: `M_schema.json`의 `importance_rules` 기준으로 규칙 기반 판정
- 0.5 미만이면 ChromaDB 저장 생략

#### `conversation/core/router.py` — `handle_turn()` 흐름
```
1. pre_process(user_input)           # 의도 분류, 명령어 감지
2. short_buf  = short_term.get(session)
3. vdb_result = long_term.query(user_input, character_id)
4. context    = prompt_build.assemble(session, short_buf, vdb_result)
5. response   = llm.generate(context)
6. post_process(session, response)   # mood/affection 업데이트, Act 전환 체크
7. session.add_turn(user_input, response)
8. summarizer.check_trigger(session) # N턴 조건 충족 시 VDB 저장
return response
```

#### `agent/state.py`
- `CH_*.yaml`의 `state.mood_triggers` 로드
- `affection` 변화 폭: 긍정 대화 `+1~3`, 부정 대화 `-1~3`, 범위 0~100 클램프
- `mood` 변화: 응답 텍스트에서 `mood_triggers` 패턴 매칭 후 세션 갱신
