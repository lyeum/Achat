### ─ Conversation 흐름 ─

```

    │
    ▼
[UI/UX] bridge.py :: sendMessage()
    │   └─ LLMWorker(QThread) 백그라운드 실행
    │
    ▼

    │
    ▼
[Conversation] ConversationRouter.handle_turn()
    │
    ├─ ① [RAG] world_nav.detect_move_intent()
    │        키워드 필터 → LLM(max_tokens=15) 장소 추출
    │        └─ 이동 감지 시: find_or_create_location()
    │               YAML acts 매칭 or LLM 새 장소 생성
    │               → VDB(world_knowledge) upsert
    │               → session.location_context 갱신
    │
    ├─ ① [NarrationMonitor] check_keyword()  ← (bridge.py 경유 선택적)
    │        키워드 감지 → narration_hardcoded.find_trigger()
    │        세션 1회 제한(_fired_keywords) → 묘사 텍스트 채팅 버블 삽입
    │
    ├─ ① [WorldTrigger] check_story_trigger / check_culture_trigger  ← (router._check_world_triggers)
    │        story: 사용자 발화 ↔ trigger_keywords 코사인 유사도 ≥ 0.55
    │             → session.fired_stories 중복 방지 → 나레이션 출력
    │        culture: LLM 의도 분류 → explained_cultures 소거법 → 미설명 항목 출력
    │        place: session.visited_places 기반 최초 진입 시 출력
    │        모두 세션 내 최초 1회 발동 (fired_stories / visited_places / explained_cultures 추적)
    │
    ├─ ② [Memory] short_term.get_recent()
    │        session.dialogue_log 슬라이딩 윈도우
    │        → Layer D (단기 히스토리 300토큰)
    │
    ├─ ③ [VDB] LongTermMemory.query()
    │        ChromaDB {char_id}_memory 컬렉션
    │        cosine 유사도 ≥ 0.52 필터
    │        → Layer C (장기 기억 150토큰)
    │
    ├─ ④ [RAG] WorldRetriever.query()
    │        ChromaDB world_knowledge 컬렉션
    │        cosine 유사도 ≥ 0.52 필터, 상위 2개
    │        → Layer B (세계관+act 200토큰)
    │
    ├─ ⑤ [Conversation] PromptBuilder.assemble()
    │        Layer A(캐릭터 시스템 300) + B + C + D
    │        → messages 리스트
    │
    ├─ ⑥ [LLM] LLMClient.generate()
    │        transformers(GPU bfloat16) or llama_cpp(CPU GGUF)
    │        repetition_penalty=1.1 / max_new_tokens=512
    │        → 캐릭터 응답 문자열
    │
    ├─ ⑦ [Conversation] update_mood() / update_affection()
    │        사용자 발화 키워드 → mood 8종 판정
    │        mood별 affection 증감 (캐릭터 YAML delta 우선)
    │
    ├─ ⑧ [Memory/VDB] summarizer.check_trigger()
    │        N턴(=5)마다: LLM 요약 → 중요도 scoring(≥0.5)
    │        mode=="function" 턴은 수집 제외 (chat 턴만 대상)
    │        → ChromaDB {char_id}_memory 저장
    │
    └─ ⑨ [학습 로그] ConversationLogger.log_turn()
             사용자 발화 키워드 카테고리 분류
             (daily/emotion/memory/advice/persona/neg/pos)
             Jaccard 중복 제거(0.55) → training/log/{cat}/*.jsonl
    │
    ▼ (응답 반환)
[UI/UX] bridge.py :: _sync_state()
    mood/배경 변화 감지 → backgroundChanged / moodChanged 시그널
    → QML CharacterDisplay / PipWindow / 배경 이미지 갱신
```

## 2. Memory / 학습 로그 → VDB 저장

### 현재 상태

**대화 메모리 (런타임)**
- **단기(short_term)**: `session.dialogue_log` 슬라이딩 윈도우 (최근 N턴). 간단하고 완전 동작.
- **장기(long_term)**: ChromaDB VDB. 요약 후 저장, 관련 입력 시 검색해 Layer C에 삽입. 완전 동작.
- **요약(summarizer)**: N턴마다 LLM으로 대화 요약 → 키워드 중요도 scoring → VDB 저장. 완전 동작.
  - **이름 importance 보완** (2026-03-28/29): `_HIGH_KEYWORDS`에 "이름:", "이름은", "이름이" 추가. 시스템 프롬프트에 "이름: [이름]" 형태 포함 지시 추가. mid 키워드 분기 dead code 수정(`score < 0.5` → `score == 0.5`).
  - 중요도 기준: high 키워드(이름/약속/갈등) → 0.85 / mid 키워드(취미/감정/날짜) → 0.6 / 무키워드 → 0.5

**학습 로그 수집 (ConversationLogger)**
- 대화 중 유저 발화 키워드를 분석해 카테고리(daily/emotion/memory/advice/persona/negative/positive) 분류.
- 트리거 조건: 키워드 매칭 즉시 flush / 반복 루프(assistant 응답 3회 동일) 즉시 flush / CHUNK_SIZE 턴 도달 / 세션 종료.
- Jaccard 유사도 기반 중복 제거 후 `training/log/{category}/YYYY-MM-DD.jsonl` 저장.
- `scripts/build_dataset.py`가 이 로그를 읽어 ChatML 포맷 학습 데이터로 변환 → LoRA 학습에 사용.

### 맞물림 이슈
- 장기 메모리 컬렉션명이 `{character_id.lower()}_memory` — 캐릭터 변경 시 새 컬렉션으로 자동 전환됨. 정상.
- 요약 트리거 간격(`memory_trigger_n=5`, 개선5 항목5 적용) — 대화 밀도에 따라 조절 여지 있음.
- **단기↔장기 경계 불명확**: 단기 버퍼(Layer D)와 장기 VDB 검색 결과(Layer C)가 중복 내용을 담을 수 있음.

### 맞물림 이슈
- **동일 ChromaDB 인스턴스 공유**: 세계관 RAG와 장기 메모리가 같은 DB를 쓰지만 컬렉션이 달라 분리는 됨. 단, DB 파일 하나가 깨지면 전체 영향.

- `ui_test` 환경에서는 임베딩 모델 None → VDB 기능 동작 안 함. 테스트 중 장기 메모리/RAG 검색 불가.

### 개선 가능성
- 세계관 RAG용 컬렉션과 메모리 컬렉션을 물리적으로 다른 DB로 분리하면 백업/마이그레이션 용이.
- threshold를 쿼리 유형(세계관 vs 메모리)별로 다르게 설정할 여지 있음 (현재 둘 다 0.52).

## 7. Tools

### 현재 상태
대화 이외의 보조 기능을 LLM 도구 호출 형식으로 제공. `agent/core.py`의 `handle_input(mode="function")`을 통해 진입하며, 슬래시 명령어(`mode="chat"`)와 별도 계층.

**트리거 방식**: 사용자 발화에서 자연어 키워드를 감지(`agent/core.py::_KEYWORDS`) → 도구 선택 → LLM이 파라미터 JSON 추출 → rule-based 실행.

> `file_convert` / `folder_classify` / `local_search` 3개 도구는 태그 클릭 즉시 다이얼로그 → bridge 슬롯 → 도구 실행으로 처리됨.
> `web_search`는 삭제됨 (2026-04-06).
>
> **개선5 항목7 (2026-04-09)**: `prompt_convert` 제외 모든 도구 진입 시 `_inject_prompt_guide(tool_name)` 호출 → ChromaDB `prompt_guides`에서 해당 모델 가이드 조회 → 시스템 프롬프트 앞단에 주입.
