## 1. assemble() 전체 조립 흐름

```mermaid
flowchart TB
    subgraph IN["외부 입력"]
        CHAR["CH_*.yaml\ncharacter dict"]
        WORLD["World_*.yaml\nworld dict"]
        SESS["ConversationSession\nmood / affection / turn_count\nsession_context / character_notes"]
        SHORT["short_buf\n단기 대화 이력"]
        VDB_IN["vdb_results\nVDB top-2 기억"]
        RAG_IN["rag_results\nRAG top-2 세계관"]
        OPS_IN["recent_ops\n기능 작업 목록 (optional)"]
        USER_IN["user_input\n현재 사용자 발화"]
    end

    subgraph LAYERS["Layer 생성"]
        LA["_layer_a()\n캐릭터 시스템 프롬프트\n≤ 300 tok"]
        LB["_layer_b()\n세계관 + Act + RAG\n≤ 200 tok"]
        LC["_layer_c()\nVDB 기억 재서술\n≤ 150 tok"]
        LE["_layer_e()\nsession_context + 약속\n≤ 200 tok"]
        LF["_layer_f()\n기능 작업 컨텍스트\n(recent_ops 있을 때만)"]
        LD["_layer_d()\n단기 히스토리\n≤ 450 tok"]
    end

    subgraph ASSEMBLE["조립 결과 messages: list[dict]"]
        SYS["messages[0]\nrole: system\n─────────────\nLayer A\n+ Layer B\n+ Layer C  (vdb_results 있을 때)\n+ Layer E  (context/notes 있을 때)\n+ Layer F  (recent_ops 있을 때)\n( .join 구분자: 빈 줄 )"]
        HIST["messages[1..N]\nrole: user / assistant\n─────────────\nLayer D\n최근 N턴 대화 쌍"]
        LAST["messages[N+1]\nrole: user\n─────────────\nuser_input"]
    end

    CHAR  --> LA
    SESS  --> LA
    WORLD --> LB
    SESS  --> LB
    RAG_IN --> LB
    VDB_IN --> LC
    CHAR  --> LC
    SESS  --> LE
    OPS_IN --> LF
    SHORT --> LD

    LA --> SYS
    LB --> SYS
    LC --> SYS
    LE --> SYS
    LF --> SYS
    LD --> HIST
    USER_IN --> LAST

    SYS  --> OUT["LLMClient.generate(messages)"]
    HIST --> OUT
    LAST --> OUT
```

## 2. Layer A — 캐릭터 시스템 프롬프트 상세

Layer A는 9개 블록을 순서대로 조립한 단일 문자열이다.

```mermaid
flowchart TD
    subgraph YAML["CH_*.yaml 슬롯"]
        Y1["name: 하루"]
        Y2["description:\n  본문 텍스트\n  예: 발화 예시 줄"]
        Y3["speech:\n  formality: 반말|존댓말\n  style: blunt|soft|직접 텍스트\n  persona: cool_observant|..."]
        Y4["personality:\n  calm|warm|energetic\n  cynical|tsundere|melancholic"]
        Y5["affection:\n  stranger: '...'\n  acquaintance: '...'\n  familiar: '...'\n  ..."]
        Y6["emotion:\n  happy: '...'\n  sad: '...'\n  annoyed: '...' ..."]
        Y7["conversation:\n  response_length: {tier: 0.0~1.0}\n  openness: {tier: 0.0~1.0}\n  directness: 0.0~1.0\n  input_length_weight: {...}"]
        Y8["rules:\n  - 규칙 문자열 목록"]
    end

    subgraph RUNTIME["런타임 상태"]
        R1["session.affection → _affection_tier()\n  0~100 값 → tier 문자열\n  (stranger/acquaintance/familiar\n   friendly/close/intimate)"]
        R2["session.mood\n  neutral|happy|sad|annoyed\n  curious|embarrassed|angry\n  affectionate|touched"]
        R3["user_input_len\n  50자 미만 → factor=0\n  200자 이상 → factor=1.0\n  → response_length / openness 완화"]
    end

    subgraph PRESETS["Preset 조회 테이블"]
        P1["_STYLE_PRESETS\nblunt → '말이 짧고 직접적'\nsoft  → '말투가 부드럽고 배려'"]
        P2["_PERSONA_PRESETS\ncool_observant / gentle_quiet\nquiet_sensitive / warm_dry"]
        P3["_PERSONALITY_PRESETS\ncalm / warm / energetic\ncynical / tsundere / melancholic"]
        P4["_AFFECTION_FALLBACK\nYAML affection 슬롯 없을 때 사용\nstranger~intimate 각 폴백 텍스트"]
        P5["_EMOTION_FALLBACK\nYAML emotion 슬롯 없을 때 사용\n8종 감정 폴백 텍스트"]
        P6["_BASE_RULES\n모든 캐릭터 강제 적용\n비한국어 문자 금지 등"]
    end

    subgraph OUTPUT_A["Layer A 출력 — 조립 순서"]
        O1["① 너는 {name}이다."]
        O2["② {description 본문}\n   [발화 패턴] 예: 줄 → 별도 블록"]
        O3["③ 반드시 반말로만 말한다. (formality)"]
        O4["④ style 지시문\n   YAML 직접 텍스트 OR preset 해석"]
        O5["⑤ persona 지시문\n   YAML 직접 텍스트 OR preset 해석"]
        O6["⑥ personality 지시문\n   YAML 직접 텍스트 OR preset 해석"]
        O7["⑦ affection tier 행동 지시문\n   YAML affection[tier] 우선\n   → 없으면 _AFFECTION_FALLBACK"]
        O8["⑧ emotion 상태 지시문\n   mood != neutral 일 때만 삽입\n   YAML emotion[mood] 우선\n   → 없으면 _EMOTION_FALLBACK"]
        O9["⑨ 대화 수위 지시문\n   response_length → 문장 수 지시\n   openness → 감정 표현 강도\n   directness → 직접성"]
        O10["⑩ [규칙]\n   _BASE_RULES + character rules"]
    end

    Y1 --> O1
    Y2 --> O2
    Y3 --> O3 & O4 & O5
    Y4 --> O6
    Y5 --> O7
    Y6 --> O8
    Y7 & R3 --> O9
    Y8 --> O10

    P1 --> O4
    P2 --> O5
    P3 --> O6
    P4 --> O7
    P5 --> O8
    P6 --> O10

    R1 --> O7 & O9
    R2 --> O8

    O1 & O2 & O3 & O4 & O5 & O6 & O7 & O8 & O9 & O10 -->|"\n".join()| RESULT["Layer A\n최종 텍스트 (~300 tok)"]
```

## 5. Layer D — 단기 히스토리 (토큰 예산 관리)

```mermaid
flowchart TD
    IN["short_buf: list[dict]\nsession.dialogue_log 전체"]

    IN --> T1["5턴 시도\nsliced = short_buf[-10:]\ntotal_tok = Σ len(content)÷2"]
    T1 --> C1{total ≤ 450 tok?}
    C1 -- Yes --> R1["5턴 반환\n10개 메시지"]
    C1 -- No --> T2["3턴 시도\nshort_buf[-6:]"]
    T2 --> C2{total ≤ 450 tok?}
    C2 -- Yes --> R2["3턴 반환\n6개 메시지"]
    C2 -- No --> T3["2턴 시도\nshort_buf[-4:]"]
    T3 --> C3{total ≤ 450 tok?}
    C3 -- Yes --> R3["2턴 반환\n4개 메시지"]
    C3 -- No --> R4["강제 2턴\nshort_buf[-2:]\n(최소 보장)"]

    R1 & R2 & R3 & R4 --> OUT["Layer D\nrole: user / assistant 교대\nmessages list에 직접 extend"]
```

## 6. 토큰 예산 전체 구조

```mermaid
%%{init: {"theme": "dark"}}%%
block-beta
  columns 1

  block:SYSTEM_MSG["system 메시지 (단일 role:system)"]
    A["Layer A\n캐릭터 페르소나\n≤ 300 tok"]
    B["Layer B\n세계관 + RAG\n≤ 200 tok"]
    C["Layer C\nVDB 기억\n≤ 150 tok"]
    E["Layer E\nsession_context\n≤ 200 tok"]
    F["Layer F\n기능 컨텍스트\n(있을 때만)"]
  end

  block:HIST_MSG["대화 이력 (role:user / assistant 교대)"]
    D["Layer D\n단기 히스토리\n≤ 450 tok\n(5→3→2턴 자동 축소)"]
  end

  block:USER_MSG["user 메시지 (role:user)"]
    U["user_input\n현재 발화"]
  end

  block:GEN["LLM 생성"]
    G["generation\n≤ 512 tok"]
  end
```

> **총 컨텍스트 예산**: 300 + 200 + 150 + 200 + 450 + 512 = **약 1,800 tok**
> `n_ctx=4096` 기준으로 여유가 있어 Layer D 초과 시 강제 축소가 거의 발생하지 않는다.

## 7. conversation 파라미터 → 지시문 변환표 (_conv_hints)

```mermaid
flowchart LR
    subgraph RL["response_length (0.0 ~ 1.0)"]
        RL1["< 0.15 → '한 문장 이내로 짧게 끊는다.'"]
        RL2["0.15~0.35 → '한두 문장 정도로 답한다.'"]
        RL3["0.35~0.55 → '두세 문장 수준으로 답한다.'"]
        RL4["0.55~0.70 → '서너 문장 정도로 답할 수 있다.'"]
        RL5["≥ 0.70 → '감정이나 생각을 여러 문장으로 표현할 수 있다.'"]
    end

    subgraph OP["openness (0.0 ~ 1.0)"]
        OP1["< 0.10 → '감정을 거의 드러내지 않는다.'"]
        OP2["0.10~0.25 → '감정을 드러내는 경우가 드물다.'"]
        OP3["0.25~0.45 → '가끔 감정이 말 속에 묻어난다.'"]
        OP4["0.45~0.65 → '감정을 자연스럽게 표현한다.'"]
        OP5["≥ 0.65 → '감정을 솔직하게 표현한다.'"]
    end

    subgraph DR["directness (0.0 ~ 1.0)"]
        DR1["< 0.30 → '말을 자주 돌려서 표현한다.'"]
        DR2["0.30~0.55 → '말을 돌려 표현하는 경우가 많다.'"]
        DR3["0.55~0.70 → '대체로 직접적으로 표현한다.'"]
        DR4["≥ 0.70 → '하고 싶은 말을 직접적으로 표현한다.'"]
    end

    subgraph ILW["input_length_weight (입력 길이 완화)"]
        ILW1["user_input_len < 50자 → factor = 0\n→ 완화 없음"]
        ILW2["50 ≤ len < 200자\n→ factor = (len-50) / 150\n→ response_length / openness 값 += weight × factor"]
        ILW3["len ≥ 200자 → factor = 1.0\n→ 최대 완화"]
    end
```

## 8. 캐릭터 YAML 슬롯 → Layer A 매핑 요약

| YAML 슬롯 | Layer A 위치 | Preset 사용 여부 | 조건 |
|---|---|---|---|
| `name` | ① 이름 | - | 있으면 항상 |
| `description` | ② 설명 / 발화 패턴 | - | 있으면 항상. `예:` 줄은 `[발화 패턴]` 블록으로 분리 |
| `speech.formality` | ③ 말투 지시 | - | `반말` / `존댓말` 특수 처리. 그 외 직접 출력 |
| `speech.style` | ④ 스타일 | `_STYLE_PRESETS` | preset key면 해석, 아니면 직접 출력 |
| `speech.persona` | ⑤ 페르소나 | `_PERSONA_PRESETS` | 동상 |
| `personality` | ⑥ 성격 | `_PERSONALITY_PRESETS` | 동상 |
| `affection[tier]` | ⑦ 친밀도 행동 | `_AFFECTION_FALLBACK` | YAML 값 우선, 없으면 폴백 |
| `emotion[mood]` | ⑧ 감정 상태 | `_EMOTION_FALLBACK` | `mood != neutral` 일 때만 |
| `conversation.*` | ⑨ 대화 수위 | - | `_conv_hints()` 수치 → 자연어 변환 |
| `rules` | ⑩ 규칙 | - | `_BASE_RULES` 앞에 항상 선행 삽입 |
