# 학습 후보 모델 목록 및 실험 설계

> 조건: 완전 로컬 (VRAM 8GB / RAM 8GB), QLoRA 학습 + LoRA 병합 + GGUF 변환 전 과정 수행 가능한 모델만 포함.
> RAM 제약으로 병합 시 FP16 CPU 로딩 한계 ≈ **3B 이하 권장, 3.8B는 위험 구간**

### 학습 → 배포 파이프라인

```
data/lora/*.jsonl
        │
        ▼  training/lora_train.py
  QLoRA 학습 (RTX 5060 Ti, VRAM 8GB)
        │  → LoRA 어댑터 저장 (adapter_model.bin)
        ▼  scripts/merge_lora.py
  LoRA 병합 (RAM ~6GB, 다른 프로세스 종료 권장)
        │  → 병합된 HF 모델 저장
        ▼  scripts/convert_to_gguf.sh
  GGUF 변환 + Q4_K_M 양자화
        │  → model.gguf (~2GB)
        ▼
  배포 환경 (Windows CPU, llama-cpp-python)
```

## 2. 실험 설계

### 2-1. 실험 흐름

```
각 모델에 대해:

① 베이스라인 측정 (파인튜닝 전)
   - 동일 테스트셋으로 한국어 품질 및 속도 측정
   - 파인튜닝 효과 비교를 위한 기준값

② QLoRA 파인튜닝
   - 동일 데이터셋 / 동일 하이퍼파라미터로 학습
   - 모델 간 공정 비교를 위해 설정값 고정

③ 파인튜닝 후 측정
   - 동일 테스트셋 재평가
   - 베이스라인 대비 개선도 측정

④ GGUF 변환 (Q4_K_M)
   - 배포 환경 속도 측정 (llama-cpp-python, CPU)

⑤ 최종 선정
   - 한국어 품질 × 응답 속도 종합 점수로 1개 모델 선정
```

### 2-2. 현행 학습 설정 (Qwen2.5-3B 기준)

```python
lora_r              = 16
lora_alpha          = 32
target_modules      = ["q_proj", "k_proj", "v_proj", "o_proj",
                        "gate_proj", "up_proj", "down_proj"]   # 전 레이어 (v1 대비 확장)
lora_dropout        = 0.1    # 과적합 억제 (v1: 0.05)
weight_decay        = 0.01

per_device_train_batch_size  = 1
gradient_accumulation_steps  = 8      # 유효 batch = 8
gradient_checkpointing       = True
max_seq_length               = 512
learning_rate                = 2e-4
optim                        = "adamw_torch"   # BitsAndBytes 미사용 (Blackwell 비지원)
bf16                         = True            # fp16 → bf16 (RTX 5060 Ti)
labels                       = "_mask_labels"  # assistant 토큰만 loss (v7~)
```

> ⚠️ **loss 스케일 변경 주의**: v7부터 assistant 토큰만 loss 계산.
> v7 이후 버전끼리만 eval_loss 값으로 직접 비교 가능.

### 2-3. 학습 데이터셋 구성

## 3. 평가 척도

### 3-1. 한국어 품질 평가 (주관 + 자동)

#### A. 이질감 점수 (자동, 낮을수록 좋음)

AI 특유의 투명 표현 빈도를 카운트한다.

```python
AI_TELL_PATTERNS = [
    "물론이죠", "물론입니다", "좋은 질문",
    "네, 알겠습니다", "도움이 되셨으면",
    "무엇을 도와드릴까요", "저는 AI",
    "제가 AI이기 때문에", "죄송합니다만",
    "말씀하신 대로", "그렇군요!"
]
# 100턴 대화에서 해당 패턴 총 등장 횟수 → 이질감 점수
```

#### B. 페르소나 일관성 점수 (주관, 1~5점)

| 점수 | 기준 |
|---|---|
| 5 | 말투/성격이 처음부터 끝까지 완전 일치 |
| 4 | 대부분 일관, 1~2회 소폭 이탈 |
| 3 | 전반적으로 유지하나 이탈이 눈에 띔 |
| 2 | 자주 캐릭터를 벗어남 |
| 1 | 페르소나 거의 미반영 |

#### C. 한국어 자연스러움 점수 (주관, 1~5점)

| 점수 | 기준 |
|---|---|
| 5 | 실제 사람 대화와 구분하기 어려움 |
| 4 | 자연스러움, 어색한 부분 거의 없음 |
| 3 | 문법적으로 맞지만 어색한 표현 있음 |
| 2 | AI 번역체 또는 딱딱한 문어체 표현 빈번 |
| 1 | 매우 어색하거나 문맥 붕괴 |

#### D. 기억 유지 정확도 (자동, %)

10턴 이후 앞선 대화 내용을 참조하는 질문에 정확히 답변하는 비율.

```
테스트 예시:
  1턴:  "내 이름은 민준이야"
  ...
  11턴: "내 이름이 뭐라고 했지?"

정답 포함 여부를 단순 키워드 매칭으로 자동 판정
```

     + (CPU 응답 속도 점수 × 0.10)

  10~15       → 4점

### 4-5. Qwen2.5-3B-Instruct LoRA v1 평가 결과 (2026-03-16)

> 학습: `training/data` 1,461건 / 3 epoch / GPU RTX 5060 (bfloat16)
> 어댑터: `output/lora_haru_v1/adapter` | 최종 loss: 0.4425 (best: 0.4267 @ epoch 2.73)

| 항목 | 베이스라인 | LoRA v1 | 변화 |
|---|---|---|---|
| AI투 표현 (건/응답) | 0.00 | 0.00 | — |
| 캐릭터 말투 (정성) | 불안정 (존댓말 혼용, 이상한 출력) | 반말·단답 일관됨 | ✅ 개선 |
| 기억 유지 정확도 | 미측정 | **40% (2/5)** | ❌ 목표 80% 미달 |
| GPU 추론 속도 | 미측정 | **10.3 tok/s** | ✅ 목표 8+ 달성 |
| CPU 속도 Q4_K_M | — | 미측정 (GGUF 변환 후) | — |

**판정:**
- ✅ 말투·페르소나: 베이스 대비 명확히 개선 (단답, 반말, 무뚝뚝한 캐릭터 표현)
- ❌ 기억 정확도: 단답 훈련으로 키워드 미포함 케이스 발생 (이름·선호·계획)
- ✅ 추론 속도: 10.3 tok/s (GPU), 목표 달성

**개선 방향:**
- 기억 참조 데이터(`memory_ref.jsonl`) 보강 — 현재 120건, 200건+ 권장

### 파일별 구현 핵심 사항

#### `training/dataset.py`
- `datasets.load_dataset("json", data_files=["data/lora/*.jsonl"])` 로드
- 각 모델의 tokenizer chat template 적용 (`tokenizer.apply_chat_template`)
- `max_length=512` 초과 샘플 필터링 및 경고 출력
- 모델별 tokenizer가 다르므로 모델명을 인자로 받아 분기 처리

#### `training/lora_train.py`
```python
# 현행 설정 (BitsAndBytes 미사용 — Blackwell SM 10.x 비지원)
lora_config = LoraConfig(
    r=16, lora_alpha=32,
    target_modules=["q_proj","k_proj","v_proj","o_proj",
                    "gate_proj","up_proj","down_proj"],  # 전 레이어
    lora_dropout=0.1, bias="none", task_type="CAUSAL_LM",
)
training_args = TrainingArguments(
    per_device_train_batch_size=1,
    gradient_accumulation_steps=8,
    gradient_checkpointing=True,
    bf16=True,                   # fp16 → bf16
    optim="adamw_torch",         # paged_adamw_8bit → adamw_torch
    output_dir="output/LoRA_vN", # training/ 아닌 output/ 하위 저장
)
# v7~: labels = _mask_labels(input_ids)  ← assistant 토큰만 loss
```
- 학습 완료 후 adapter 저장: `output/LoRA_vN/adapter/`
- 체크포인트는 학습 완료 후 자동 삭제 (adapter/만 유지)

#### `scripts/merge_lora.py`
```python
# RAM 6GB 이내 병합을 위한 설정
model = AutoModelForCausalLM.from_pretrained(
    base_model_path,
    torch_dtype=torch.float16,
    low_cpu_mem_usage=True,   # RAM 절약 핵심
    device_map="cpu",

model = PeftModel.from_pretrained(model, adapter_path)
model = model.merge_and_unload()
model.save_pretrained(output_path)
tokenizer.save_pretrained(output_path)
```
- 실행 전 다른 프로세스 종료 권장 (RAM 여유 확보)
- OOM 발생 시 swap 공간 4GB 임시 확장 후 재시도

| 기억 유지 정확도 | `eval/memory_test.py` | 자동 |
| GPU 속도 | `eval/speed_bench.py` (transformers) | 자동 |
| CPU 속도 Q4_K_M | `eval/speed_bench.py` (llama-cpp) | 자동 |

**`eval/ai_tell_checker.py` 핵심 로직:**
```python
# 섹션 3-1.A의 AI_TELL_PATTERNS 기준
def count_ai_tells(responses: list[str]) -> int:
    return sum(
        1 for r in responses
        for pattern in AI_TELL_PATTERNS
        if pattern in r
    )
```

**`eval/memory_test.py` 핵심 로직:**
```
1. 고정 시나리오 대화 10턴 진행 (테스트셋 섹션 4-1 기준)
2. 11턴째에 기억 참조 질문 입력
3. 응답에 정답 키워드 포함 여부 판정
4. 정확도 = 정답 수 / 전체 기억 참조 질문 수
```
