# BUG_04.md — 한국어 특화 베이스 모델(Qwen2.5-3B-Korean) 채택 실험 및 반려

> 작성일: 2026-04-03
> 상태: 반려 (v10 vanilla 채택)
> 목적: 베이스 모델을 한국어 특화 버전으로 교체했을 때 캐릭터 품질 개선 여부 검증

---

## 1. 배경

LoRA_v9까지 `Qwen/Qwen2.5-3B-Instruct` 기반으로 학습했으나, 조사/어미 오류 및 비자연스러운 한국어 표현이 잔존.
한국어 특화 베이스 모델(`MyeongHo0621/Qwen2.5-3B-Korean`)을 적용하면 베이스 수준의 한국어 이해가 올라와
캐릭터 LoRA가 말투/성격 학습에 더 집중할 수 있지 않을까 하는 가설로 실험 진행.

## 3. 실제 대화 비교 결과

직접 UI 대화 테스트(`uv run python main.py`) 및 `chat_cli.py` 비교 테스트 진행.

### LoRA_v10 (vanilla)
- 캐릭터 말투 일관성 유지 — 단답, 반말, 감정 억제 패턴 자연스러움
- 이전 버전(v9) 대비 전반적 응답 품질 향상
- 특이 패턴 없음

### LoRA_v10_korean
- **입력 재조립 패턴 발생**: 유저 발화를 그대로 반복하거나 재조립한 후 응답하는 경향
  ```
  User: 오늘 좀 피곤한데
  하루: 오늘 피곤하다고 했지? 그래.   ← 재조립 후 응답
  ```
- 베이스 모델(200k 어시스턴트 스타일)의 "확인 → 응답" 패턴이 캐릭터 LoRA에 의해 완전히 덮이지 않음
- eval_loss도 v10 대비 높음 (1.563 vs 1.512) — 수렴도 더 낮음

## 4. 원인 분석

### 핵심 문제: 학습량 비대칭

```
베이스 모델 학습량:  200,000건 (어시스턴트 대화 스타일)
캐릭터 LoRA 학습량:    2,150건 (캐릭터 스타일)
비율: 약 1:93
```

2,150건의 캐릭터 데이터로 200,000건으로 형성된 베이스 응답 패턴을 덮어쓰기에는 학습량이 압도적으로 부족.

### 강한 규제가 해결책이 되지 않는 이유

입력 재조립 패턴을 없애려면 매우 높은 lora_dropout, 강한 EWC 패널티, 더 많은 epoch이 필요.
그러나 이 조건에서는 캐릭터 데이터 과적합이 먼저 발생하고 베이스 스타일이 그대로 잔존하는 악순환.
결과적으로 학습 비용 대비 효과가 없음.

### vanilla Instruct가 더 유리한 이유

`Qwen/Qwen2.5-3B-Instruct`는 다국어 사전학습 모델이라 한국어 기초는 이미 내재.
200k 어시스턴트 스타일 추가 파인튜닝 없이 캐릭터 LoRA만 얹는 구조가 오히려 간섭이 적음.

## 6. 향후 학습 방향성

### 단기 — v11 (현실적)

- LoRA_v10 기반 EWC Fisher 계산 후 feedback_pos 누적 데이터 추가 학습
- memory_test 테스트 케이스 질문 모호성 수정 완료 후 재측정
- 목표: memory_test 4/5 이상 안정적 달성

### 중기 — DPO

- feedback_pos/neg 쌍이 각 20건 이상 누적 시 DPO 파이프라인 구축
- SFT(LoRA)로 기본기 → DPO로 잘못된 응답 패턴 핀포인트 교정
- 입력 재조립, 문법 오류, AI투 표현 잔존 케이스를 rejected로 활용

### 장기 — 한국어 특화 베이스 재검토 조건

한국어 특화 베이스 모델은 다음 조건이 충족됐을 때 재검토:

1. 캐릭터 학습 데이터 **10,000건 이상** 확보 (현재 2,150건)
2. 재조립 패턴을 DPO rejected 데이터로 충분히 수집
3. 또는 베이스 모델 자체가 어시스턴트 스타일이 약한 순수 instruct 모델일 때

현재 단계에서는 vanilla Instruct + 캐릭터 LoRA 조합이 최적.
