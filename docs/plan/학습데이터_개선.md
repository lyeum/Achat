## 배경 및 문제 인식

### 현재 구조의 문제

```
training/data/emotion/angry.jsonl  → "현재 화가 난 상태. 말이 차갑고 날카로워진다."
CH_Haru.yaml: state.tone_guide     → 친밀도별 톤만 정의 (감정은 없음)
prompt_build.py: _MOOD_HINT        → 감정 상수가 YAML 밖 코드에 하드코딩
```

- 학습 데이터 카테고리와 캐릭터 YAML 필드가 **서로 다른 언어**로 분리되어 있음
- 감정(emotion), 성격(personality), 말투(speech_style) 정의가 코드에 산재
- 신규 캐릭터 추가 시 어느 필드를 어떻게 채워야 하는지 명확한 계약이 없음
- 학습 데이터 시스템 프롬프트(BASE + 카테고리 속성) vs 추론 시 프롬프트(description + speech + tone + mood + conv_hints) **형식 불일치**

### 목표 구조

```
training/data/
  affection/*    →  character YAML의 affection: 슬롯
  emotion/*      →  character YAML의 emotion: 슬롯
  speech_style/* →  character YAML의 speech: 슬롯
  personality/*  →  character YAML의 personality: 슬롯
```

카테고리 = 캐릭터 스키마 슬롯의 어휘(vocabulary).
새 캐릭터는 각 슬롯에 preset 이름 또는 직접 기술 텍스트를 채워서 정의한다.

## Step 1 — 캐릭터 스키마 정의

**파일:** `conversation/character/character_schema.yaml` (신규)

캐릭터 정의 파일의 계약(contract) 역할. 모든 CH_*.yaml이 이 스키마를 따른다.

```yaml
# ── 기본 정보 ──────────────────────────────────────────────────────────────
name: ""           # (필수) 캐릭터 이름
description: ""    # 캐릭터 외형/성격 개요 (자유 기술, 추론 시 시스템 프롬프트에 삽입)

# ── 말투 슬롯 (speech_style/* 카테고리 대응) ──────────────────────────────
speech:
  formality: ""    # 반말 | 존댓말
  style: ""        # preset: blunt | soft  /  또는 직접 기술 텍스트
  persona: ""      # preset: cool_observant | gentle_quiet | quiet_sensitive | warm_dry
                   #         또는 직접 기술 텍스트

# ── 성격 유형 슬롯 (personality/* 카테고리 대응) ──────────────────────────
personality: ""    # preset: calm | cynical | tsundere  /  또는 직접 기술 텍스트

# ── 친밀도 tier별 행동 슬롯 (affection/* 카테고리 대응) ───────────────────
affection:
  stranger: ""
  acquaintance: ""
  familiar: ""
  friendly: ""
  close: ""
  intimate: ""

# ── 감정 상태별 행동 슬롯 (emotion/* 카테고리 대응) ───────────────────────
emotion:
  neutral: ""      # 기본 상태 (보통 생략)
  happy: ""
  sad: ""
  angry: ""
  annoyed: ""
  curious: ""
  embarrassed: ""
  touched: ""
  affectionate: ""

# ── 규칙 ──────────────────────────────────────────────────────────────────
rules: []          # 위반 금지 항목 리스트

# ── 상태 초기값 및 전환 임계값 ──────────────────────────────────────────────
state:
  mood_default: neutral
  affection_default: 30
  affection_thresholds:
    stranger:     [0,  15]
    acquaintance: [16, 30]
    familiar:     [31, 50]
    friendly:     [51, 70]
    close:        [71, 85]
    intimate:     [86, 100]
  mood_triggers: {}
  affection_delta: {}

# ── 대화 수위 파라미터 ────────────────────────────────────────────────────
conversation:
  response_length: {}   # tier별 0.0~1.0
  openness: {}          # tier별 0.0~1.0
  directness: 0.5       # 고정값 0.0~1.0

# ── 기억 표현 방식 ─────────────────────────────────────────────────────────
memory_voice: ""        # VDB 검색 결과 재서술 시 참고 힌트
```

**preset 해석 테이블** (prompt_build.py에 상수로 보유):

| 슬롯 | preset 키 | 해석 텍스트 |
|---|---|---|
| speech.style | blunt | 말이 짧고 직접적이다. 불필요한 설명이나 완충 표현을 하지 않는다. |
| speech.style | soft | 말투가 부드럽고 배려가 있다. 상대의 반응을 살피며 말한다. |
| speech.persona | cool_observant | 감정을 억제하고 상황을 관찰하는 말투. 반응이 냉정하고 분석적이다. |
| speech.persona | gentle_quiet | 조용하고 온화한 말투. 상대를 배려하며 천천히 말한다. |
| speech.persona | quiet_sensitive | 말수가 적지만 상대의 감정에 민감하게 반응한다. |
| speech.persona | warm_dry | 따뜻하지만 표현이 건조하다. 직접적이지 않게 감정을 전달한다. |
| personality | calm | 차분하고 안정된 태도. 쉽게 흔들리지 않는다. |
| personality | cynical | 냉소적이고 현실주의적이다. 세상을 비관적으로 보지만 날카로운 통찰이 있다. |
| personality | tsundere | 츤데레 성격이다. 직접적으로 감정을 표현하지 않고 부정하거나 딴 말을 하지만 행동으로 드러난다. |
| personality | energetic | 활발하고 주도적이다. 반응이 빠르고 주변 분위기를 이끄는 경향이 있다. |
| personality | melancholic | 감수성이 깊고 내면적이다. 감정을 직접 말하기보다 분위기나 표현으로 전달한다. |
| personality | warm | 따뜻하고 다정하다. 상대의 감정을 먼저 헤아리고 자연스럽게 배려한다. |

## Step 2 — CH_Haru.yaml 재작성

**파일:** `conversation/character/CH_Haru.yaml`

스키마 슬롯을 기준으로 재작성. 기존 필드 중 스키마에 없는 것은 제거 또는 통합.

주요 변경:
- `speech_style` (monolithic) → `speech.formality` / `speech.style` / `speech.persona`로 분리
- `state.tone_guide` → 최상위 `affection:` 슬롯으로 이동
- 감정 행동 텍스트 추가 (`emotion:` 슬롯 — 현재 prompt_build.py 내부 `_MOOD_HINT`에만 있음)
- `personality.emotional_expression` (수치) → `personality` 텍스트 슬롯으로 통합

## Step 3 — prompt_build.py 개편

**파일:** `conversation/core/prompt_build.py`

`_layer_a()` 를 스키마 슬롯 기반 조립으로 교체.

```
조립 순서:
  너는 {name}이다.
  {description}
  {speech.formality}을 사용한다.
  {speech.style}      ← preset 해석 또는 직접 텍스트
  {speech.persona}    ← preset 해석 또는 직접 텍스트
  {personality}       ← preset 해석 또는 직접 텍스트
  {affection[tier]}   ← 현재 tier의 행동 텍스트
  {emotion[mood]}     ← mood != neutral일 때만 삽입
  {conv_hints}        ← response_length / openness / directness → 자연어
  {rules}             ← 각 항목을 문장으로
```

삭제 대상:
- `_TONE_DEFAULT` 상수 → `affection:` 슬롯 폴백으로 대체
- `_MOOD_HINT` 상수 → `emotion:` 슬롯 폴백으로 대체

## Step 4 — 학습 데이터 시스템 프롬프트 갱신

**파일:** `training/data/**/*.jsonl` (전체 3,170개 샘플, 57파일, v11 기준)

현재 학습 데이터 형식:
```
너는 {char_name}이다. [범용 제약 5문장]
[카테고리 속성 1문장]
```

Step 3 완료 후 추론 시 조립 결과와 형식을 일치시키기 위해 갱신.

각 파일 카테고리에 따라 조립 결과의 관련 슬롯만 포함한 형식으로 업데이트:
```
너는 {char_name}이다. {speech} {personality}
[affection or emotion 카테고리 속성]
```

갱신 스크립트: `training/scripts/rewrite_system_prompts.py` 수정하여 재실행.

```
반말을 사용한다.

차분하고 안정된 태도. 쉽게 흔들리지 않는다.
조금 편해진 상태. 가끔 관심이 묻어나오지만 여전히 담담하다.

불필요한 인사말이나 AI 투 표현을 사용하지 않는다. 올바른 한국어 문법을 사용한다.
```
