## UI-1. 슬라이더 이벤트 탈취 수정 ✅ 완료

### 문제

ScrollView 내 `Slider` 컴포넌트는 Flickable이 수평 드래그 이벤트를 가로채
실제로 핸들이 움직이지 않는 현상이 발생했다.  
또한 window-level `DragHandler`가 비활성화되지 않아 창 전체가 움직이는 문제도 있었다.

### 원인

| 문제 | 원인 |
|---|---|
| 창 전체 이동 | `DragHandler.enabled`가 `adminPanelOpen / charCreateOpen` 조건을 누락 |
| 슬라이더 무반응 | `ScrollView` Flickable이 수평 드래그를 먼저 소비 |

### 수정

- `main.qml` — `DragHandler.enabled: !root.adminPanelOpen && !root.charCreateOpen`  
- `AdminPanel.qml`, `CharacterCreatePanel.qml` — `ScrollView`에 `contentWidth: availableWidth` 추가 (수평 Flickable 비활성화)

#### QtQuick.Controls Slider 완전 교체

`Slider` 컴포넌트 자체를 제거하고, `MouseArea { preventStealing: true }`를 사용한  
커스텀 슬라이더(Rectangle + 드래그 MouseArea)로 전체 교체.

**대상:**

| 파일 | 슬라이더 |
|---|---|
| `ui_ux/qml/AdminPanel.qml` | 친밀도 슬라이더 (`affTrack`) |
| `ui_ux/qml/AdminPanel.qml` | 직접성 슬라이더 (`drTrack`) |
| `ui_ux/qml/CharacterCreatePanel.qml` | 직접성 슬라이더 (`drTrackC`) |

**구현 패턴:**
```qml
Item {
    id: drTrack
    Layout.fillWidth: true; height: 20
    property real drVal: ...
    property bool _dragging: false

    // 트랙 + 핸들 Rectangle ...

    MouseArea {
        anchors.fill: parent
        preventStealing: true          // ← 핵심
        cursorShape: Qt.SizeHorCursor
        onPressed:  drTrack._dragging = true
        onReleased: { drTrack._dragging = false; /* signal emit */ }
        onPositionChanged: {
            if (pressed) {
                var v = Math.round(mouseX / drTrack.width * 20) / 20
                drTrack.drVal = Math.max(0.0, Math.min(1.0, v))
            }
        }
    }
}
```

## UI-2. 응답길이 · 감정개방도 토글 버튼 오류 수정 ✅ 완료

### 문제

`AdminPanel.qml:273`, `CharacterCreatePanel.qml:398` 에서  
`TypeError: Cannot read property '2' of undefined` 오류 발생.  
토글 버튼을 눌러도 tier별 값이 변경되지 않았다.

### 원인

QML Repeater delegate는 시각적 부모(visual parent)에 삽입된다.  
중첩 Repeater에서 내부 delegate의 `parent` 체인:

```
MouseArea (내부 Repeater delegate 내)
  parent → Rectangle (내부 delegate)
  parent.parent → RowLayout (내부 Repeater의 부모)
  parent.parent.parent → 외부 Item delegate  ← 여기
  parent.parent.parent.parent → Column (contentCol)  ← 잘못 참조됨
```

기존 코드 `parent.parent.parent.parent._rlVals[index]`는
Column에서 `_rlVals`를 찾으려 했기 때문에 `undefined` 오류.

`sel` 프로퍼티의 `parent.parent.parent.selIdx()`도 동일하게 한 단계 초과.

### 수정

외부 `Item` delegate에 `property int tierIndex: index` 명시 + `id` 부여,  
내부에서 해당 id를 직접 참조:

```qml
// 외부 Repeater delegate
Item {
    id: rlRow
    property int tierIndex: index    // ← 명시적 property로 캡처
    readonly property var _rlVals: [0.2, 0.5, 0.8]
    function selIdx() { ... }

    RowLayout {
        Repeater {
            model: ["짧게", "보통", "길게"]
            Rectangle {
                property bool sel: rlRow.selIdx() === index  // id 직접 참조
                MouseArea {
                    onClicked: {
                        var tier = adminRoot._tiers[rlRow.tierIndex]
                        var val  = rlRow._rlVals[index]
                        ...
                    }
                }
            }
        }
    }
}
```

**대상:**

| 파일 | 섹션 | id |
|---|---|---|
| `ui_ux/qml/AdminPanel.qml` | 응답 길이 | `rlRow` |
| `ui_ux/qml/AdminPanel.qml` | 감정 개방도 | `opRow` |
| `ui_ux/qml/CharacterCreatePanel.qml` | 응답 길이 | `rlRowC` |
| `ui_ux/qml/CharacterCreatePanel.qml` | 감정 개방도 | `opRowC` |

## UI-3. CharacterStatusPanel 하단 오버플로 수정 ✅ 완료

하단 텍스트가 모달 밖으로 삐져나옴.

                 statusRoot.height - 96)

## UI-4. MemoryDBPanel 세계관 폼 Placeholder 텍스트 ✅ 완료

### 배경

세계관 항목 추가 폼의 `TextInput` / `TextEdit` 필드에 작성 예시가 없어  
어떤 형식으로 입력해야 하는지 불명확했다.

### 수정

`TextInput`은 `placeholderText` 미지원 (`TextInput`은 QtQuick 기본 컴포넌트,
`placeholderText`는 `TextField`/Qt6 전용). **Text 오버레이 방식** 적용.

```qml
Rectangle { width: 120; ... clip: true
    TextInput { id: wAddWorldId; ... }
    Text {
        anchors { left: parent.left; verticalCenter: parent.verticalCenter; leftMargin: 6 }
        text: "ex) seaside_world"; color: "#505070"; font.pixelSize: 10
        visible: wAddWorldId.text.length === 0 && !wAddWorldId.activeFocus
    }
}
```

**대상 필드:**

| id | placeholder |
|---|---|
| `wAddWorldId` | `ex) seaside_world` |
| `wAddSection` | `place / story` |
| `wAddTitle` | `ex) 등대지기 전설` |
| `wAddTrigger` | `ex) 등대, 전설, 노인` |
| `wAddContent` | 내용 TextEdit 오버레이 |

**파일:** `ui_ux/qml/MemoryDBPanel.qml`

## DB-1. CRUD 편집 영역 — 스크롤 가능한 TextEdit ✅ 완료

### 문제

장기기억·세계관 RAG의 내용 편집 영역(`TextEdit` 60~64px 고정 높이)에서  
텍스트가 길어지면 잘려 보이고 스크롤이 불가능했다.

### 수정

`TextEdit`를 `Flickable` 래퍼 안에 배치, 높이를 100px로 증가:

```qml
Rectangle {
    width: parent.width; height: 100; radius: 4
    color: "#181828"; border.color: "#3A3A60"; clip: true

    Flickable {
        anchors.fill: parent; anchors.margins: 6
        contentWidth: width
        contentHeight: editContent.implicitHeight
        clip: true; flickableDirection: Flickable.VerticalFlick
        ScrollBar.vertical: ScrollBar { policy: ScrollBar.AsNeeded }

        TextEdit {
            id: editContent; width: parent.width
            color: "#E0E0E0"; font.pixelSize: 11
            wrapMode: TextEdit.Wrap; selectByMouse: true
        }
    }
}
```

**대상:**

| 파일 | 영역 |
|---|---|
| `ui_ux/qml/MemoryDBPanel.qml` | 장기기억 수정 (`editContent`) |
| `ui_ux/qml/MemoryDBPanel.qml` | 세계관 RAG 수정 (`wEditArea`) |

## DB-2. 세계관 RAG 트리 구조 표기 ✅ 완료

### 변경 전

```
▸ seaside_world / culture (3개)
▸ seaside_world / place (5개)
▸ seaside_world / story (6개)
```

`world_id + section` 플랫 그룹으로 나열 → 세계관이 여러 개일 때 가독성 저하.

### 변경 후

```
▸ seaside_world (14개)
    ▸ culture (3개)
        [청크 카드]
    ▸ place (5개)
    ▸ story (6개)
```

### 구현

`_worldGroups` 계산 로직을 2단계 중첩 맵으로 변경:

```javascript
// world_id → { section → chunks[] } 계층 맵
var groups = [{
    world_id: "seaside_world",
    total: 14,
    sections: [
        { section: "culture", chunks: [...] },
        { section: "place",   chunks: [...] },
        { section: "story",   chunks: [...] }
    ]
}]
```

QML 렌더링: 외부 `Repeater`(world_id) → 내부 `Repeater`(section) → 청크 카드.

**파일:** `ui_ux/qml/MemoryDBPanel.qml`

## F-1. 세계관 생성 패널 (WorldCreatePanel) ✅ 완료

### 배경

기존 DB 패널의 "항목 추가" 폼은 세계관의 단일 항목만 추가 가능했다.  
세계관 전체를 한 번에 작성하려면 culture/place/story 섹션별로 여러 항목을  
반복 입력해야 하는 불편함이 있었다.

### 구현

신규 파일 `ui_ux/qml/WorldCreatePanel.qml`:

**구조:**
- `world_id` 입력 필드 (필수)
- culture / place / story 3개 섹션 블록
- 각 섹션: 초기 1개 항목, `+ 추가` 버튼으로 최대 4개까지 확장
- 각 항목: 제목(TextInput), 내용(Flickable TextEdit 72px), 트리거 키워드(TextInput)
- 저장 버튼: 제목·내용 모두 입력된 항목만 `bridge.addWorldKnowledge()` 호출

**QML 컴포넌트 구조:**
```
WorldCreatePanel
  └─ component SectionBlock   (culture/place/story 섹션 블록)
       └─ component ItemForm  (항목 입력 폼 — 제목/내용/트리거)
```

**데이터 흐름:**
```
저장 버튼 클릭
  → SectionBlock.getItem(i) 로 각 항목 데이터 수집
  → 제목·내용 비어있지 않은 항목만
  → bridge.addWorldKnowledge(world_id, section, title, content, trigger)
  → 패널 닫기
```

**메뉴 진입:**
- `SettingsPanel.qml` — "세계관" 섹션 최상단에 `+ 세계관 생성` 버튼 추가
- `SettingsPanel` — `signal worldCreateRequested()` 추가
- `main.qml` — `property bool worldCreateOpen`, `onWorldCreateRequested` 핸들러, `WorldCreatePanel` 오버레이(z: 25)

**수정/신규 파일:**

| 파일 | 내용 |
|---|---|
| `ui_ux/qml/WorldCreatePanel.qml` | **신규** — 세계관 생성 패널 |
| `ui_ux/qml/SettingsPanel.qml` | `worldCreateRequested` signal + `+ 세계관 생성` 버튼 |
| `ui_ux/qml/main.qml` | `worldCreateOpen` 프로퍼티 + `WorldCreatePanel` 오버레이 |
