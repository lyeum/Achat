## 1. 문제 진단

### 1-1. CH_default.yaml 경고

```
WARNING | [character_load] 누락 필드 ['name', 'speech_style', 'memory_voice', 'state'] — CH_default.yaml
```

- `CH_default.yaml`은 기본값 제공용 파일인데 `CH_*.yaml` 패턴에 매칭되어 캐릭터로 취급됨
- `REQUIRED_FIELDS` 검사에 걸려 매 실행마다 경고 4회 발생
- `CharacterSelectPanel`에서도 기본 캐릭터가 목록에 노출됨

### 1-2. 대화 품질 문제 (feedback_neg 기준)

수집된 `training/log/feedback_neg/` 데이터에서 반복되는 패턴:

| 증상 | 예시 | 빈도 |
|---|---|---|
| 문법 오류 | "원래 이렇아." (존재하지 않는 조사) | 높음 |
| 외국어 단어 혼입 | "이상함axed." | 낮음 |
| stranger tier 부자연스러움 | 첫 인사에 "어색해?" 반문 | 중간 |
| 긴 대화 후 말투 붕괴 | 10턴 이후 일관성 감소 | 중간 |

### 1-3. 이름 기억 실패

- `memory_test.py` 결과: 4/5 (이름 기억 FAIL)
- `score_importance()` 기준: "이름" 키워드가 `_HIGH_KEYWORDS`에 있으나, 요약 LLM이 "이름"이라는 단어를 생략하고 요약하면 importance=0.5로 낮게 산정됨
- 검색 threshold=0.7 (cosine), 이름 관련 메모리가 임계값을 못 넘는 경우 발생

**파일:** `ui_ux/bridge.py`
**완료:** 2026-03-28 · CharacterSelectPanel에서 "default" 항목 미표시

#### C. `CH_Haru.yaml` — speech_style / rules 강화 ✅

**speech_style에 추가:**
```yaml
반드시 올바른 한국어 문법을 사용한다. 한국어가 아닌 다른 언어의 단어를 문장에 섞지 않는다.
잘못된 예: "원래 이렇아." → 올바른 예: "원래 이래."
잘못된 예: "이상함axed." → 올바른 예: "그냥 이상한 거야."
처음 만난 상대에게는 먼저 감정적 질문을 하지 않는다.
```

**rules에 추가:**
```yaml
- 한국어가 아닌 다른 언어(영어, 일본어 등)의 단어를 문장 중간에 섞지 않는다
- 문법에 맞지 않는 조사 결합을 사용하지 않는다
- 처음 만난 상대(stranger tier)에게 먼저 감정적 질문을 던지지 않는다
```

**파일:** `conversation/character/CH_Haru.yaml`
**완료:** 2026-03-28

#### D. `summarizer.py` — 이름 포함 발화 importance 보완 ✅

요약 시스템 프롬프트에 지시 추가:
```python
"사용자의 이름이 대화에서 언급된 경우, 반드시 '이름: [이름]' 형태로 요약에 포함해. "
```

`_HIGH_KEYWORDS`에 변형 패턴 추가:
```python
_HIGH_KEYWORDS = [
    "이름:", "이름은", "이름이", "이름",  # "이름: X" 형태 우선 인식
    ...
]
```

**파일:** `memory/summarizer.py`
**완료:** 2026-03-28

### [즉시, 추가 발견] 코드 정합성 수정

#### BUG-17. `prompt_build.py` — YAML rules 무시 (학습/추론 불일치) ✅

C 항목 완료 후 정합성 검토에서 발견. `_layer_a()`가 실제 YAML rules를 무시하고 하드코딩된 요약을 사용하고 있었음.
- `build_sft_from_feedback.py`는 YAML rules를 직접 join → 학습 데이터와 추론 프롬프트 불일치
- 이름 형식도 `"너의 이름은 X이다."` vs `"너는 X이다."` 불일치

**수정:** `_layer_a()`에서 문자열 rules 리스트를 직접 join, 이름 형식 통일
**파일:** `conversation/core/prompt_build.py`
**완료:** 2026-03-29 · 참조: [BUG_small-clear.md BUG-17](../BUG/BUG_small-clear.md)

#### BUG-18. `summarizer.py` — mid 키워드 분기 죽은 코드 ✅

D 항목 완료 후 검토에서 발견. `if score < 0.5:` 조건이 항상 False → mid 키워드(취미/감정) 분기가 실행 안 됨.

**수정:** `if score == 0.5:` 로 변경
**파일:** `memory/summarizer.py`
**완료:** 2026-03-29 · 참조: [BUG_small-clear.md BUG-18](../BUG/BUG_small-clear.md)
