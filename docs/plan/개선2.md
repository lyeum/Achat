## 공통 전략: Windows / WSL2 / Linux 조건부 동작

모든 파일·URL 열기 동작은 플랫폼마다 처리 방식이 다르다.

```python
import platform, os

def _is_wsl() -> bool:
    return platform.system() == "Linux" and "microsoft" in platform.uname().release.lower()

def _is_windows() -> bool:
    return platform.system() == "Windows"
```

| 환경 | 파일 열기 | URL 열기 |
|---|---|---|
| Windows native | `os.startfile(path)` | `os.startfile(url)` 또는 `subprocess ["start", url]` |
| WSL2 | `powershell.exe -NoProfile -Command "Invoke-Item 'PATH'"` | `cmd.exe /c start "" "https://..."` |
| Linux / macOS | `subprocess ["xdg-open", path]` | `subprocess ["xdg-open", url]` |

## 1. 파일이름 변경 — 한국어 마지막 글자 씹힘

### 근본 원인

QML `TextInput`의 `onTextChanged`는 IME 입력 조합 중(preedit 상태)에도 발화된다.
한국어 입력 시 마지막 글자가 조합 중인 상태에서 "적용" 버튼을 클릭하면:

1. `onTextChanged` → `_renameTo = "이름"` (조합 완료 전 상태)
2. 클릭 이벤트 → `_renameTo` 에는 이미 이전 값이 남아 마지막 글자 누락

### 수정 방법

`FileOptionsPanel.qml`의 적용 버튼 `onClicked` 에서 `fileOptRoot._renameTo` 대신
`renameInput.text`를 직접 읽어 사용한다.

**수정 전:**
```qml
onClicked: {
    var renameTo = (fileOptRoot._mode === 0) ? fileOptRoot._renameTo : ""
    ...
}
```

**수정 후:**
```qml
onClicked: {
    var renameTo = (fileOptRoot._mode === 0) ? renameInput.text.trim() : ""
    ...
}
```

또한 `TextInput`의 `onTextChanged` 핸들러와 `_renameTo` 바인딩은 제거해도 무방하다.
(적용 버튼에서만 값을 읽으면 되기 때문)

### 대상 파일

| 파일 | 변경 |
|---|---|
| `ui_ux/qml/FileOptionsPanel.qml` | 적용 버튼 `onClicked`에서 `renameInput.text` 직접 참조 |

### 테스트

## 2. 파일검색 — PNG 파일 열기 실패

### 근본 원인

`bridge.py`의 `openFile()` 슬롯이 `subprocess.run(["cmd.exe", "/c", "start", "", path])` 방식을 사용한다.
WSL2 환경에서 파일 경로는 `/home/...` 또는 `\\wsl.localhost\...` UNC 형식인데,
`cmd.exe /c start` 는 UNC 경로를 지원하지 않아 PNG·이미지 파일 열기가 실패한다.

### 수정 방법

플랫폼 감지 후 분기:

```python
def _open_file(path: str) -> None:
    if _is_windows():
        os.startfile(path)
    elif _is_wsl():
        # UNC 경로를 지원하는 Invoke-Item 사용
        subprocess.Popen(
            ["powershell.exe", "-NoProfile", "-Command", f"Invoke-Item '{path}'"],
            stdout=subprocess.DEVNULL, stderr=subprocess.DEVNULL
        )
    else:
        subprocess.Popen(["xdg-open", path])
```

`bridge.py`의 `openFile` 슬롯에서 위 함수를 호출한다.

### 대상 파일

| 파일 | 변경 |
|---|---|
| `ui_ux/bridge.py` | `openFile()` 슬롯 — 플랫폼별 분기 함수 적용 |

### 테스트

- `TestLocalSearchFullFlow::test_open_file_wsl_powershell` — WSL2 모킹 후 `powershell.exe Invoke-Item` 호출 확인
- `TestLocalSearchFullFlow::test_open_file_windows_startfile` — Windows 모킹 후 `os.startfile` 호출 확인

## 3-A. 웹검색 — URL 브라우저 열기 실패

### 근본 원인

`ChatBubble.qml`에서 `onLinkActivated: Qt.openUrlExternally(link)` 를 사용하는데,
WSL2 환경에서 Qt는 브라우저를 감지하지 못해 `unable to detect a web browser` 오류가 발생한다.

HTTP URL은 `cmd.exe /c start "" "https://..."` 방식으로 열면 WSL2에서도 정상 작동한다
(UNC 경로 문제가 없는 일반 URL이기 때문).

### 수정 방법

**QML 측:** `onLinkActivated`에서 `Qt.openUrlExternally` 대신 `bridge.openUrl(link)` 호출

```qml
onLinkActivated: function(link) {
    bridge.openUrl(link)
}
```

**bridge.py 측:** `openUrl` 슬롯 추가

```python
@Slot(str)
def openUrl(self, url: str) -> None:
    if _is_windows():
        os.startfile(url)
    elif _is_wsl():
        subprocess.Popen(
            ["cmd.exe", "/c", "start", "", url],
            stdout=subprocess.DEVNULL, stderr=subprocess.DEVNULL
        )
    else:
        subprocess.Popen(["xdg-open", url])
```

### 대상 파일

| 파일 | 변경 |
|---|---|
| `ui_ux/bridge.py` | `openUrl(url: str)` 슬롯 추가 |
| `ui_ux/qml/ChatBubble.qml` | `onLinkActivated` → `bridge.openUrl(link)` |

### 테스트

- `TestWebSearchFullFlow::test_link_click_opens_browser_wsl` — WSL2 모킹 후 `cmd.exe /c start` 호출 확인

## 4-A. 해시태그 — 선택 즉시 다이얼로그 열기

### 근본 원인

현재 파일 다이얼로그 호출이 `sendMessage()` 내부에 위치해 있어,
사용자가 태그를 선택하고 메시지를 전송해야 비로소 파일 선택 창이 열린다.
자연스러운 UX는 태그 선택 즉시 파일 창이 열리는 것이다.

### 수정 방법

`main.qml`의 태그 선택 핸들러(`onCurrentTagChanged` 또는 태그 클릭 `onClicked`)에서
파일 다이얼로그를 즉시 호출한다.

```qml
onClicked: {
    currentTag = tag.key
    if (tag.key === "file_convert" || tag.key === "folder_classify" || tag.key === "local_search") {
        bridge.browseFiles(tag.key)   // 즉시 호출
    }
}
```

`sendMessage()` 내부의 중복 다이얼로그 호출 로직은 제거한다.

### 대상 파일

| 파일 | 변경 |
|---|---|
| `ui_ux/qml/main.qml` | 태그 클릭 핸들러에 즉시 다이얼로그 호출 이동 |

### 테스트

## 5. 캐릭터 교체 버그 + 통합 테스트

### 근본 원인 (추정)

`bridge.py`의 `changeCharacter()` 슬롯:

```python
@Slot(str)
def changeCharacter(self, char_id: str) -> None:
    if getattr(self._agent, "_stub", True):   # ← stub일 때만 실행?
        return
    self._agent.change_character(char_id)
```

`_stub` 가드 조건이 반전되어 있을 가능성이 높다.
실제 에이전트(`_stub=False`)일 때도 `return`이 실행되어 캐릭터 교체가 동작하지 않는다.

### 수정 방법

```python
if getattr(self._agent, "_stub", False):  # stub이면 early return
    return
```

또는 `agent/core.py`에서 `_stub` 속성 선언 방식을 확인 후 조건 정정.
