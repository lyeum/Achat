## 항목 1 — aff 역할 재정의: emotion 강도 조절자 ✅ 완료 (2026-04-16)

### 현재 구조의 문제

```
affection 45 → familiar tier → "조금 편해진 상태. 가끔 관심이 묻어나오지만 여전히 담담하다."
affection 51 → friendly tier → "마음을 조금 열기 시작한 상태. ..."
```

- tier 경계(예: 50→51)에서 행동 묘사 텍스트가 통째로 교체 → 표현이 뚝 끊김
- affection 슬롯이 "감정 상태별 행동 규칙" 역할 → emotion 슬롯과 중복·충돌
- aff 변화가 emotion에 의존(`affection_delta`) → 역방향 의존성 누적

### 목표 구조

```
aff = "이 캐릭터가 상대에게 감정을 얼마나 솔직하게 드러내는가"의 강도 조절자

tier   낮음(stranger/acquaintance): 감정을 억제, 표현이 건조·무뚝뚝
tier   중간(familiar/friendly):     감정이 자연스럽게 묻어남
tier   높음(close/intimate):        감정 표현이 즉각적·솔직, 숨기지 않음

특수 이벤트(발작버튼/고백/배신): aff 강제 점프 + emotion 강제 전환
일반 상호작용: 정수(+1~+3) 점진 변화 — 급격한 변화 없음
```

#### 1-D. `agent/state.py` — `check_trigger_events()` 함수 추가

```python
def check_trigger_events(
    session: ConversationSession,
    user_input: str,
    character: dict,
) -> bool:
    """trigger_events 키워드 감지 → aff 점프 + mood 강제 전환.

    발동하면 True 반환 (일반 update_mood/update_affection 건너뜀).
    cooldown_turns 동안 동일 이벤트 재발동 방지.
    """
    events: dict = character.get("state", {}).get("trigger_events", {})
    if not events:
        return False

    fired = getattr(session, "_fired_events", {})   # {event_name: remaining_turns}
    # cooldown 감소
    fired = {k: v - 1 for k, v in fired.items() if v - 1 > 0}

    for event_name, cfg in events.items():
        if event_name in fired:
            continue
        keywords: list[str] = cfg.get("keywords", [])
        if not any(kw in user_input for kw in keywords):
            continue

        # aff 변경
        if "aff_set" in cfg:
            old = session.affection
            session.affection = max(0, min(100, cfg["aff_set"]))
            logger.info(f"[trigger] {event_name}: aff {old} → {session.affection} (set)")
        elif "aff_delta" in cfg:
            old = session.affection
            session.affection = max(0, min(100, session.affection + cfg["aff_delta"]))
            logger.info(f"[trigger] {event_name}: aff {old} → {session.affection} (delta)")

        # mood 강제 전환
        if "mood" in cfg:
            old_mood = session.mood
            session.mood = cfg["mood"]
            logger.info(f"[trigger] {event_name}: mood {old_mood} → {session.mood}")

        # cooldown 등록
        cooldown = cfg.get("cooldown_turns", 0)
        if cooldown > 0:
            fired[event_name] = cooldown

        session._fired_events = fired
        return True

    session._fired_events = fired
    return False
```

## 항목 2 — `**...**` 나레이션 박스 렌더링 ✅ 완료 (2026-04-16)

### 현재 상태 분석

| 경로 | 상태 | 비고 |
|---|---|---|
| LLM 응답 파싱 | ✅ 구현 | `_on_response()` → `_split_narration()` |
| 사용자 입력 파싱 | ✅ 구현 | `sendMessage()` → `_split_narration()` |

Connections {
    target: bridge
    function onMessageReplaced(index, segments) {
        // 기존 버블 제거
        messageModel.remove(index, 1)
        // 분할 버블 삽입 (역순으로 insert → 순서 유지)
        for (var i = segments.length - 1; i >= 0; i--) {
            messageModel.insert(index, {
                role:    segments[i].role,
                content: segments[i].content
            })
        }
    }
}

## 항목 3-2 — 데이터 품질 검증 🔲 고려 중

### 문제 정의

- `review.py`의 y/n 판정이 주관적 기준에만 의존 — 재현 불가
- 학습 데이터 JSONL 파일 간 품질 편차가 측정 안 됨
- "좋은 데이터"의 정량적 기준이 없어 신규 데이터 추가 시 품질 보장 불가

### 구현 방향

#### 3-2-A. 품질 지표 정의

| 지표 | 측정 방법 | 목표 기준 |
|---|---|---|
| `ai_tell` | AI투 표현 키워드 수 / 응답 수 | 0.0 (이상적) |
| `persona_hit` | 페르소나 키워드 매칭 비율 | tier별 기대 키워드 포함률 ≥ 0.3 |
| `length_ok` | tier별 기대 길이 범위 내 비율 | ≥ 0.7 |
| `jaccard_max` | 파일 내 최대 Jaccard 중복도 | ≤ 0.55 (이미 적용) |
| `review_pass` | review.py y 판정 비율 (있을 때) | — 기준선 수집용 |

#### 3-2-B. `training/scripts/quality_score.py` 신규

```
입력:  training/data/**/*.jsonl (또는 단일 파일)
출력:  파일별 지표 테이블 + 전체 요약
       → 기준 미달 파일 목록 + 개선 권고
```

#### 3-2-C. `review.py` 통합

review.py에서 y 판정 시 해당 샘플의 지표를 함께 기록 → 기준선 자동 갱신.

### 전제 조건
