# LoRA 학습 실행 가이드

> 환경: Ubuntu 24.04 WSL2 / RTX 5060 Ti (VRAM 8GB) / CUDA 12.8
> 모델: Qwen2.5-3B-Instruct
> ⚠️ BitsAndBytes 4-bit 양자화 미사용 (Blackwell SM 10.x 미지원) — bfloat16 풀 파라미터 + LoRA

---

## 전체 흐름

```
training/log/*.jsonl          (수동 수집 대화 데이터)
    │
    ▼  scripts/build_dataset.py
data/lora/conversation/       (시스템 프롬프트 삽입 + ChatML 포맷)
    │
    ├─ data/lora/function/    (기능 모드 예시 데이터 — 이미 작성됨)
    │
    ▼  training/lora_train.py
output/{run_name}/adapter/    (LoRA 어댑터)
    │
    ▼  eval/
평가 결과 출력
```

## Step 1. 대화 데이터 수집

`training/log/` 폴더에 카테고리별 JSONL 파일에 대화를 추가합니다.
형식은 `training/log/_schema.json` 참조.

```jsonl
# training/log/advice.jsonl 예시 (system 메시지 없이 저장)
{"messages": [{"role": "user", "content": "이직할까 말까 고민이야."}, {"role": "assistant", "content": "왜, 거기서 뭔 일 있어?"}], "character_id": "Haru", "category": "advice", "affection": "mid", "mood": "neutral", "emotion_trigger": "고민_상담"}
```

| 카테고리 | 파일 | 최소 목표 |
|---|---|---|
| 일상 잡담 | `daily.jsonl` | 30건 |
| 감정 반응 | `emotion.jsonl` | 30건 |
| 고민 상담 | `advice.jsonl` | 30건 |
| 기억 참조 | `memory.jsonl` | 20건 |
| 페르소나 유지 | `persona.jsonl` | 20건 |

> `training/data/` 의 기존 데이터(common/, personality/, affection/, speech_style/)는
> 시스템 프롬프트가 이미 포함된 완성 데이터 — build_dataset.py 없이 직접 학습에 사용 가능.

## Step 2. 데이터 빌드 (training/log → data/lora/conversation)

```bash
# 기본 실행 (dry-run으로 먼저 검증)
uv run python scripts/build_dataset.py --dry_run

# 실제 빌드
uv run python scripts/build_dataset.py

# 옵션 확인
uv run python scripts/build_dataset.py --help
```

**주요 옵션:**
| 옵션 | 기본값 | 설명 |
|---|---|---|
| `--log_dir` | `training/log` | 입력 JSONL 디렉토리 |
| `--out_dir` | `data/lora/conversation` | 출력 디렉토리 |
| `--char_dir` | `conversation/character` | 캐릭터 YAML 디렉토리 |
| `--dry_run` | — | 출력 없이 검증만 실행 |

빌드 후 확인:
```bash
# 출력 파일 확인
ls data/lora/conversation/

# 데이터 구조 검증 (토크나이저 없이)
uv run python training/dataset.py --data_dir data/lora/conversation
```

## Step 4. LoRA 학습

> 디바이스 자동 선택: CUDA 사용 가능 → GPU (bfloat16), 없으면 → CPU (float32) 자동 전환

### 4-A. GPU 학습 (정식)

```bash
uv run python training/lora_train.py \
  --model Qwen/Qwen2.5-3B-Instruct \
  --data_dir data/lora \
  --output_dir output/lora_haru_v1 \
  --epochs 3
```

### 4-B. CPU 파이프라인 테스트 (저장 없이)

```bash
# training/data/ 기존 데이터로 5스텝만 실행 — 파이프라인 동작 확인용
uv run python training/lora_train.py \
  --model Qwen/Qwen2.5-3B-Instruct \
  --data_dir training/data \
  --no_save \
  --max_steps 5 \
  --max_length 128 \
  --logging_steps 1
```

> ⚠️ CPU 모드 메모리 주의:
> - Qwen2.5-3B float32 로드: RAM **~12GB** 필요
> - 부족 시 1.5B 모델 사용: `--model Qwen/Qwen2.5-1.5B-Instruct` (~6GB)
> - 또는 WSL2에서 `wsl --shutdown` 후 재시작해 메모리 확보

**주요 옵션:**
| 옵션 | 기본값 | 설명 |
|---|---|---|
| `--model` | `Qwen/Qwen2.5-3B-Instruct` | HuggingFace 모델명 |
| `--data_dir` | `data/lora` | 학습 데이터 루트 |
| `--output_dir` | `output/lora_v1` | 어댑터 저장 경로 |
| `--subset` | `None` (전체) | `conversation` \| `function` |
| `--epochs` | `3` | 학습 에폭 수 |
| `--max_steps` | `-1` (에폭 기준) | 양수 설정 시 해당 스텝에서 종료 |
| `--no_save` | — | 학습 후 어댑터 저장 건너뜀 |
| `--batch_size` | `1` | 배치 크기 |
| `--grad_accum` | `8` | Gradient accumulation (유효 배치=8) |
| `--lr` | `2e-4` | 학습률 |
| `--max_length` | `512` | 최대 시퀀스 길이 |
| `--lora_r` | `16` | LoRA rank |
| `--lora_alpha` | `32` | LoRA alpha |

**GPU OOM 발생 시:**
```bash
uv run python training/lora_train.py \
  --model Qwen/Qwen2.5-3B-Instruct \
  --data_dir data/lora \
  --output_dir output/lora_haru_v1 \
  --epochs 3 \
  --grad_accum 16 \
  --max_length 256
```

**학습 중 VRAM 모니터링 (별도 터미널):**
```bash
watch -n 2 nvidia-smi
```

학습 완료 후 어댑터 위치: `output/lora_haru_v1/adapter/`
