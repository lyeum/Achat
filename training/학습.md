# LoRA 학습 매뉴얼

> 환경: Ubuntu 24.04 WSL2 / RTX 5060 Ti (VRAM 8GB / RAM 8GB) / CUDA 12.8
> 모델: Qwen2.5-3B-Instruct
> 방식: bfloat16 풀 파라미터 + LoRA (BitsAndBytes 4-bit 미사용 — Blackwell SM 10.x 미지원)

---

## 1. 학습 루프 내부 흐름

```
[데이터 로드]
training/data/**/*.jsonl (또는 data/lora/)
    │
    ▼ load_training_data()
HuggingFace Dataset 객체 (text 필드)
    │
    ▼ tokenize_dataset()   — truncation=True, max_length=512
    │                        labels = input_ids (전체 토큰 loss)
    │
    ▼ train_test_split()   — eval_split > 0 이면 90% train / 10% eval 분리
    │                        eval_split = 0 이면 전체 train, eval 없음
    │
[모델 로드]
AutoModelForCausalLM (bfloat16, device_map="auto")
    │
    ▼ get_peft_model()     — LoRA 어댑터 주입
    │                        target: q/k/v/o/gate/up/down proj (전 레이어)
    │                        trainable params: ~30M / 전체 3.1B
    │
[학습 루프 — HuggingFace Trainer]
    │
    ├─ 매 스텝:
    │   forward → loss 계산 → backward → gradient accumulation(8스텝)
    │   → optimizer step (AdamW + cosine LR decay)
    │
    ├─ logging_steps마다:  train loss / grad_norm / lr / epoch 출력
    │
    ├─ save_steps(100)마다:
    │   eval_split > 0 → eval loss 계산 → best이면 checkpoint 교체
    │   eval_split = 0 → checkpoint만 저장 (eval 없음)
    │
    └─ 학습 완료:
        load_best_model_at_end=True (eval 있을 때) → best checkpoint 복원
        model.save_pretrained() → output/{run_name}/adapter/ 저장
```

**총 스텝 수 계산:**
```
총 스텝 = (샘플 수 × 0.9) ÷ (batch_size × grad_accum) × epochs
예 | 1461 × 0.9 ÷ (1 × 8) × 4 ≈ 660 스텝
```

## 2. 학습 시작 방법

### 2-1. 사전 확인

```bash
# VRAM 여유 확인 (학습 전 잔류 프로세스 정리)
nvidia-smi

# 잔류 Python 프로세스 확인 및 정리
ps aux | grep lora_train | grep -v grep
kill <PID>

# 패키지 확인
uv run python -c "import peft, transformers, datasets; print('OK')"
```

### 2-2. 백그라운드 실행 (권장)

```bash
mkdir -p training/LoRA_v4  #FIX# 버전명 — 로그 파일 저장을 위해 먼저 생성
cd /home/trusia/projects/Achat && nohup uv run python training/lora_train.py \
  --model Qwen/Qwen2.5-3B-Instruct \
  --data_dir training/data \
  --output_dir training/LoRA_v4 \  #FIX# 버전명 (예: LoRA_v5)
  --epochs 4 \                     #FIX# epoch 수
  --weight_decay 0.01 \
  --lora_dropout 0.1 \
  --eval_split 0.1 \
  --logging_steps 20 \
  > training/LoRA_v4/train.log 2>&1 &  #FIX# 버전명 일치
echo "PID: $!"
```

> ⚠️ **eval_split 주의**: VRAM 8GB 환경에서 eval 시점에 gradient_checkpointing 비활성화 →
> activation 전체 VRAM 점유로 `CUDA error: unknown error` 발생 가능.
> 발생 시 `--eval_split 0` 으로 끄거나 다른 프로세스를 먼저 모두 정리한 후 재실행.

## 3. 실시간 모니터링

tqdm 출력은 `\r` 덮어쓰기 방식이라 로그 파일에서 grep이 안 됨.
**checkpoint의 trainer_state.json을 직접 읽는 방식** 사용 (100 스텝마다 갱신).

### loss 그래프 (자동 생성)

학습 중 **epoch 종료마다** + **전체 학습 완료 시** 자동으로 PNG 저장됨:

```
training/LoRA_v4/         #FIX# 버전명
├─ loss_curve_epoch01.png   ← epoch 1 종료 시
├─ loss_curve_epoch02.png   ← epoch 2 종료 시
├─ ...
└─ loss_curve_final.png     ← 전체 완료 시 (train + eval 통합)
```

- train loss: 파란 실선
- eval loss: 빨간 실선 + 원 마커 (eval_split > 0 일 때만)
- best eval 지점: 점선 수직선 표시

### 진행도 + train loss (checkpoint 기반)

```bash
ls -d training/LoRA_v4/checkpoint-* 2>/dev/null | sort -t- -k2 -n | tail -1 | \  #FIX# 버전명
xargs -I{} python3 -c "
import json, sys
s = json.load(open('{}/trainer_state.json'))
logs = [l for l in s['log_history'] if 'loss' in l and 'eval' not in str(l)]
print(f'진행: {s[\"global_step\"]} / {s.get(\"max_steps\",\"?\")} step')
print(f'best eval loss: {s.get(\"best_metric\", \"없음 (eval 비활성)\")}')
print()
for l in logs[-5:]:
    print(f'  step {l[\"step\"]:>4}  epoch {l[\"epoch\"]:.2f}  loss {l[\"loss\"]}')
"
```

## 4. 인자 전체 목록

| 인자 | 기본값 | 설명 |
|---|---|---|
| `--model` | `Qwen/Qwen2.5-3B-Instruct` | HuggingFace 모델명 |
| `--data_dir` | `data/lora` | 학습 데이터 루트 디렉토리 |
| `--output_dir` | `output/lora_v1` | 체크포인트 및 어댑터 저장 경로 |
| `--subset` | `None` (전체) | `conversation` / `function` / `None`(전체 혼합) |
| `--epochs` | `3` | 학습 에폭 수 |
| `--batch_size` | `1` | 디바이스당 배치 크기 |
| `--grad_accum` | `8` | Gradient accumulation 스텝 수 (유효 배치 = batch×grad_accum) |
| `--lr` | `2e-4` | 초기 학습률 (cosine decay 적용) |
| `--max_length` | `512` | 최대 시퀀스 길이 (초과 시 잘림) |
| `--lora_r` | `16` | LoRA rank (높을수록 파라미터 많음, 표현력 증가) |
| `--lora_alpha` | `32` | LoRA alpha (스케일링 = alpha/r, 통상 r의 2배) |
| `--lora_dropout` | `0.05` | LoRA dropout 비율 (과적합 억제 시 `0.1` 권장) |
| `--weight_decay` | `0.0` | L2 정규화 강도 (과적합 억제 시 `0.01` 권장) |
| `--eval_split` | `0.1` | 검증셋 비율 (`0` 이면 eval 완전 비활성) |
| `--max_samples` | `-1` (전체) | 학습에 사용할 최대 샘플 수 (shuffle 후 선택) |
| `--save_steps` | `100` | 체크포인트 저장 및 eval 실행 주기 (스텝 단위) |
| `--logging_steps` | `10` | train loss 출력 주기 (스텝 단위) |
| `--max_steps` | `-1` (에폭 기준) | 양수 설정 시 해당 스텝 수에서 강제 종료 |
| `--no_save` | `False` | 학습 후 어댑터 저장 건너뜀 (파이프라인 테스트용) |

## 5. 과적합 분석 및 대응

### 학습 루프의 과적합 유발 요소

| 요소 | 현재 기본값 | 영향 | 권장 |
|---|---|---|---|
| `weight_decay` | `0.0` | L2 정규화 없음 → 가중치 무제한 성장 | `0.01` |
| `lora_dropout` | `0.05` | 소규모 데이터에서 너무 낮음 | `0.1` |
| `labels = input_ids` | 전체 토큰 loss | 시스템 프롬프트·유저 턴까지 학습 → 불필요한 암기 | assistant 토큰만 (현재 미구현) |
| 샘플 수 1,461건 | 고정 | 적은 데이터 × 많은 epoch = 빠른 암기 | `--max_samples` 로 조절 |

**실제 관측 (lora_haru_v3, 30 epoch):**
```
epoch  3   eval 0.7226  ← best
epoch  5   eval 0.79    ↑ 과적합 시작
epoch 30   train 0.07 / eval 1.02  ↑ 완전 암기
```
→ **유효 학습 구간은 epoch 3~4** — 이후는 train loss만 낮아지고 eval loss 반등

### 샘플링 동작 방식

`--max_samples N` 지정 시 전체 데이터에서 무작위 N건만 추출.

```
전체 데이터 로드
    │
    ▼  shuffle(seed=42)
    │
    ▼  select(range(N))
    │
    ├─ 90%  train_ds
    └─ 10%  eval_ds
```

> `seed=42` 고정 — 같은 `--max_samples` 값이면 항상 동일한 샘플 선택.
